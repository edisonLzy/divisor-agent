## 2. 原型设计（Wireframes）

基于需求文档的 UI 界面原型与交互说明。

### 2.1 全局布局（主界面）

采用经典的 C/S 桌面应用布局（如类似 Cursor / Chat 界面），左侧为会话树管理，右侧为对话与操控区。

```text
+-------------------------------------------------------------------------+
|  [+] 新建会话树 (New)                           [⚙️ 本地模型/环境设置]  |
+--------------------------+----------------------------------------------+
| 🌲 会话树 (Session Tree)  | 💬 聊天界面 (Chat Area)                      |
|                          |                                              |
| 🔍 搜索历史会话...        | [Agent] 10:42 AM                             |
|                          | ▶ 思考过程 (Thinking...) [点击展开]          |
| 🔽 默认工作区             | ------------------------------------------   |
|  ├── 📄 初始化项目        | 我已经为您读取了 `server/index.ts`。         |
|  │   ├── 📄 增加API      | 基于您的需求，建议对 WebSocket 握手进行以下修改：|
|  │   │   └── 📄 调试Bug  | ...                                          |
|  │   └── 📄 重构代码 (当前)|                                              |
|  ├── 📄 Rust ACP 联调     | [User] 10:45 AM                              |
|                          | 那就按照建议修改吧，注意兜底异常。           |
|                          |                                              |
|                          | [Agent] 10:45 AM                             |
|                          | ⏳ 正在请求执行文件写入操作...               |
|                          | [ ✓ 文件已修改 ]                             |
|                          |                                              |
|                          |                        [🔀 一键从当前派生]   |
|                          +----------------------------------------------+
|                          | 📝 告诉 Agent 下一步做什么...                |
|                          | [ 📎 添加上下文 ]                 [ 发送 ↗ ] |
+--------------------------+----------------------------------------------+
```

### 2.2 核心功能与交互说明

#### 2.2.1 会话树与分支管理（Tree Management）

- **展示**：左侧边栏以树状结构（父子关系）展示所有的历史对话，每个节点包含 `title` 和 `timestamp`。
- **当前定位**：当前正在进行的会话分支会有明显的高亮选中态。
- **一键 Fork（全局派生）**：在聊天界面底部，提供「一键从当前派生」按钮。点击后，基于当前完整的链路（从根节点到当前叶子），复制出一个新的子节点，并且在此独立沙箱中继续对话。

#### 2.2.2 消息流与流式渲染（Stream & Thinking）

- **增量渲染**：对话内容必须支持 `text_delta` 打字机渲染，首 token 响应 < 2s。
- **思考折叠（`thinking_delta`）**：在普通文字输出前，展示 Agent 的推理和规划过程。界面默认将其装载在可折叠的容器中（`▶ 思考过程`），保持主界面干净，用户可点击展开排障。

#### 2.2.3 POI 断点派生（POI Fork）

当用户对 Agent 的某段特有回答（或代码片段）有疑问，想专门针对这句话展开新话题，而不引起原会话的主线发散时：

```text
+------------------------------------------------------------+
| [Agent]                                                    |
| 推荐边界: 历史数据读取优先走 HTTP；仅实时对话走 ACP...     |
|                                                            |
|      +------------------------------------------+          |
|      | 🖱️ 选中句子后右键 / 悬浮菜单              |          |
|      | ---------------------------------------- |          |
|      | 📋 复制 (Copy)                           |          |
|      | 🔀 以此为断点派生 (Fork from this point)  | <------- |
|      | 📌 设为书签 (Bookmark)                    |          |
|      +------------------------------------------+          |
+------------------------------------------------------------+
```

- **交互**：选中片段（POI）或在对应消息气泡上右键，弹出菜单。选择「以此为断点派生」，前端提取包含该文本的 `messageId`，通过 ACP 请求服务端将其作为新会话树的起点，界面自动切换至新建的树节点。

#### 2.2.4 人机协作授权审批弹窗（HITL Auth Modal）

当核心大脑（服务端）通过 ACP 请求执行本地高风险操作（写文件、执行 Shell 命令）时，Tauri (Rust) 强制拦截并弹出系统级/应用级模态审批窗口：

**终端命令审批**：

```text
+-------------------------------------------------------------+
| ⚠️ 本地操作授权请求 (Action Required)                         |
|-------------------------------------------------------------|
| Agent 正在请求访问您的本地系统以执行操作，请确认：              |
|                                                             |
| 🚨 操作类型：终端命令执行 (terminal/create)                   |
| 💻 请求命令：`rm -rf ./temp_build`                            |
| 📂 工作目录：/Users/zhiyu/Desktop/coding/divisor-agent/      |
|                                                             |
| 风险提示：此命令匹配到高风险操作黑名单，执行后不可撤销。        |
|                                                             |
| [ ❌ 拒绝 (Reject) ]                   [ ✅ 批准 (Approve) ] |
+-------------------------------------------------------------+
```

**文件修改审批**：

```text
+-------------------------------------------------------------+
| ⚠️ 本地文件写入审批                                           |
|-------------------------------------------------------------|
| 📂 目标路径：.../divisor-agent/packages/server/src/index.ts  |
|                                                             |
| 📄 差异预览 (Diff)：                                         |
| ---------------------------------------------------------   |
|   // original                                               |
|- const wsPort = 8080;                                      |
|   // modified                                               |
|+ const wsPort = process.env.VITE_WS_PORT || 8080;          |
| ---------------------------------------------------------   |
|                                                             |
| [ ❌ 拒绝写入 ]                               [ ✅ 允许写入 ] |
+-------------------------------------------------------------+
```

- **审批反馈**：
  - 点击"批准"：触发 IPC 授信给 Rust 执行命令或落盘操作，结果反馈给 Node。
  - 点击"拒绝"：Rust 返回 `permission_denied` 给 Node，并在聊天流中展示出"用户已拒绝该操作"。
