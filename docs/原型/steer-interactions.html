A. Assistant Steer Event · Goal

目标方案。steer 是当前 assistant 生成过程的 UI 控制事件：pending 只表示未消费；消费后从 pending 移除，并出现在当前 assistant response 的消费位置。

消息语义

  • Steer：影响当前运行，不开启新 user/assistant turn。
  • Follow-up：排队到当前完成后，开启下一轮 user/assistant。
  • Thinking：只展示模型真实 thinking，不混入用户 steer。

落点判断

AssistantMessage UI event
  • 不是 ThinkingContent：steer 来自用户，不是模型内部推理。
  • 不是 TextContent：不应进入复制、fork、历史正文。
  • 不放在 processing 顶部：应按被消费时的 response offset 插入正文流。

需要改的 UI 规则

  • pending 表示“未消费”，被消费后必须从 pending 区域移除。
  • 移除后记录当前 assistant 已输出文本长度，按 response offset 插入。
  • 输入框上方可短暂显示 recently applied，随后只在 assistant 内长期保留。

事件流示意

user prompt append message entry，启动 assistant streaming
steerPrompt add pending prompt，runtime 消费后绑定到 current assistant entry metadata
assistant continues 同一个 assistant message 继续更新，不新建回复
