The current sticky wrapper sits inside a virtual row that is absolutely positioned and moved with transform. The sticky element can only stick inside that row's containing block, which is roughly the height of the message.

Once the row is outside the virtual range, React Virtual can unmount it. So a nested sticky child has neither enough containing space nor stable DOM lifetime.

Processed 9s

The clean fallback is a sibling overlay inside ChatMessages. The scrollable virtual list keeps its current absolute row structure. The overlay listens to virtualizer state and renders the active user prompt independently.

On every scroll or virtualizer change, read scrollOffset and the measurements cache. Walk the user-message indexes and select the last one whose measured end is before the viewport top. That entry becomes the active sticky prompt.

The overlay can reuse the user bubble styling, but it should avoid joining the measured virtual row tree. Actions can be overlay-local: copy text, scrollToIndex for jump, and optionally request edit mode on the source row.
