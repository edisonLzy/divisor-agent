# 前端技术文档（MVP）

## 11. 技术栈

| 层级 | 技术 |
| :--- | :--- |
| UI 框架 | React 19 + TypeScript |
| 宿主容器 | Tauri v2 Webview |
| 与 Rust 通信 | Tauri IPC（`invoke` / `listen`，`@tauri-apps/api` v2） |
| 与服务端通信 | tRPC v11 + `@trpc/react-query`（`@tanstack/react-query` v5） |
| 构建工具 | Vite 7 |
| 样式 | Tailwind CSS v4（`@tailwindcss/vite` 插件集成，无 `tailwind.config.js`） |
| 开发端口 | 1420（`strictPort: true`，端口占用直接报错）|

---

## 2. 职责边界

前端（Webview JS）负责：
- 所有 UI 渲染：消息流、会话树、审批弹窗、POI 选区菜单。
- 通过 **tRPC** 读写**元数据**（会话列表、历史消息、设置）。
- 通过 Tauri IPC 向 Rust Core 发起指令（开始会话、Fork、审批决策）。
- 监听 Rust Core 推送的事件（消息流、权限申请）。

前端**不**负责：
- 直接访问本地文件系统或终端。
- 维护 ACP WebSocket 连接（由 Rust 负责）。

## 3. 页面与组件规划

### 3.1 整体布局

```
┌──────────────┬─────────────────────────────────┐
│  SessionTree │         ChatView                │
│  (侧边栏)     │  ┌─────────────────────────────┐│
│              │  │  MessageList (滚动区)         ││
│              │  │  ┌───────────────────────┐   ││
│              │  │  │ AssistantMessage       │   ││
│              │  │  │  ThinkingBlock(折叠)   │   ││
│              │  │  │  TextBlock(流式)       │   ││
│              │  │  └───────────────────────┘   ││
│              │  └─────────────────────────────┘│
│              │  InputBar (输入框 + Fork 按钮)    │
└──────────────┴─────────────────────────────────┘
```

### 3.2 SessionTree（会话树侧边栏）

**数据来源：** `trpc.sessions.list.useQuery()`

**节点数据结构：**
```ts
interface SessionNode {
  id: string;
  parentId: string | null;
  name: string;
  timestamp: number;
  children?: SessionNode[];
}
```

**交互要求：**
- 递归渲染树节点，支持展开/收起。
- 当前活跃节点高亮。
- 支持重命名（`trpc.sessions.rename.useMutation()`）。
- 支持删除（`trpc.sessions.delete.useMutation()`）。
- 初始加载 + mutation 后自动 invalidate 缓存刷新。
- 性能要求：1k 节点以内首屏渲染 < 1.5 秒。

### 3.3 ChatView（消息视图）

**数据来源（初始加载）：** `trpc.sessions.history.useQuery({ id: sessionId, cursor })`（支持游标分页）

**实时消息：** 监听 Rust IPC 事件 `agent_message_chunk`

#### 3.3.1 消息类型渲染

| 消息类型 | 渲染方式 |
| :--- | :--- |
| `text_delta` | 增量追加到当前 AssistantMessage 文本块，实时渲染 |
| `thinking_delta` | 追加到 ThinkingBlock，默认折叠，可手动展开 |
| 用户消息 | 静态气泡渲染 |
| 工具调用结果 | 可折叠的代码块或结构化展示 |

#### 3.3.2 流式渲染要求
- 首 token 可见时间目标 < 2 秒。
- 文本增量追加时不触发整页重渲，使用局部更新策略（如 `useRef` + DOM 直接写入或细粒度状态切片）。
- 会话中断后重连时，自动通过 HTTP 拉取最新历史块补齐。

#### 3.3.3 POI 选区交互
- 每条 AssistantMessage 段落绑定 `messageId`。
- 用户选中文本后展示右键菜单，包含「以此为断点派生」选项。
- 触发后调用 IPC：`invoke('session_fork', { sessionId, messageId })`。

### 3.4 InputBar（输入栏）

- 多行文本输入框，支持 `Shift+Enter` 换行，`Enter` 发送。
- 「发送」按钮：调用 `invoke('session_prompt', { sessionId, content })`。
- 「全局 Fork」按钮：调用 `invoke('session_fork', { sessionId })` 不传 `messageId`，复制完整链路创建新分支。
- 发送中状态：禁用输入框与发送按钮，显示 loading 状态。

### 3.5 ApprovalDialog（审批弹窗）

**触发方式：** 监听 Rust IPC 事件 `session_request_permission`

**弹窗内容：**
- 操作类型（文件写入 / 终端执行 / 跨目录访问）。
- 完整操作参数（路径、命令、Diff 内容等）。
- 「批准」和「拒绝」两个操作按钮。

**审批决策调用：**
```ts
// 批准
invoke('permission_approve', { requestId })
// 拒绝
invoke('permission_reject', { requestId })
```

**交互约束：**
- 弹窗出现时，对话输入区必须被禁用（等待用户决策）。
- 弹窗为模态，不可跳过。

## 4. IPC 接口清单

> 说明：与服务端的元数据通信通过 tRPC 完成（见第 5 节），本节仅列与 Rust 的 IPC 接口。

### 4.1 前端 → Rust（invoke）

| 命令名 | 参数 | 说明 |
| :--- | :--- | :--- |
| `session_start` | `{ sessionId: string }` | 建立 ACP 连接，开始会话 |
| `session_prompt` | `{ sessionId: string, content: string }` | 发送用户消息 |
| `session_fork` | `{ sessionId: string, messageId?: string }` | 全局 Fork 或 POI Fork |
| `permission_approve` | `{ requestId: string }` | 批准高风险操作 |
| `permission_reject` | `{ requestId: string }` | 拒绝高风险操作 |

### 4.2 Rust → 前端（listen 事件）

| 事件名 | 载荷 | 说明 |
| :--- | :--- | :--- |
| `agent_message_chunk` | `{ type: 'text_delta' \| 'thinking_delta', delta: string, sessionId: string }` | 流式消息块 |
| `session_request_permission` | `{ requestId: string, operation: string, params: object }` | 高风险操作审批请求 |
| `session_forked` | `{ newSessionId: string }` | Fork 完成通知 |
| `acp_error` | `{ code: string, message: string }` | ACP 连接或执行错误 |

## 6. 状态管理设计

```
AppState
├── sessions: SessionNode[]       // 会话树（来自 HTTP）
├── activeSessionId: string       // 当前激活会话
├── messages: Map<sessionId, Message[]>  // 各会话消息缓存
├── streamingState: { sessionId, isStreaming, pendingChunk }
└── pendingApproval: ApprovalRequest | null  // 当前待审批请求
```

- 使用 React Context + useReducer 管理（项目未引入额外状态管理库）。
- `messages` 按 sessionId 分片缓存，切换会话时优先读缓存，无缓存时走 HTTP 拉取。
- 流式 chunk 合并在内存中进行，结束后写入 messages 缓存。

## 7. 错误处理

| 错误场景 | 处理方式 |
| :--- | :--- |
| HTTP 请求失败 | Toast 提示 + 允许重试 |
| ACP 连接断开（`acp_error`） | 状态栏显示「连接断开」，自动重连由 Rust 负责，前端监听恢复事件 |
| 审批超时（用户未操作） | 保持弹窗展示，不自动处理 |
| Fork 失败 | 弹窗提示错误信息 |

## 8. 验收要求（前端视角）

- [ ] 会话树正确递归渲染，切换节点加载对应历史消息。
- [ ] `text_delta` 流式增量渲染，UI 无跳动。
- [ ] `thinking_delta` 块默认折叠，可点击展开。
- [ ] 审批弹窗在收到 `session_request_permission` 后正确弹出，批准/拒绝后弹窗关闭。
- [ ] 写操作被拒绝后对话可继续（不崩溃）。
- [ ] POI 右键菜单在选中文本时出现，触发 Fork 后侧边栏新增节点。
- [ ] 全局 Fork 按钮成功创建新分支并跳转。
- [ ] 首屏会话树加载 < 1.5 秒（1k 节点），首 token 可见 < 2 秒。
