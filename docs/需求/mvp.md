# Pi Agent 原生应用 MVP 需求文档

## 12. 后续演进（非 MVP）
### 5.2 模块 B: 树状会话管理（Tree Management）

#### 5.2.1 会话树结构
- 侧边栏递归展示会话节点，节点至少包含 `id`、`parentId`、`name`、`timestamp`。
- 节点支持展开/收起、选中高亮、当前分支定位。

#### 5.2.2 元数据同步
- 前端通过 HTTP `GET /api/sessions` 拉取会话树。
- 前端支持轮询或事件触发刷新，保证会话树与服务端一致。

#### 5.2.3 历史会话回放
- 点击历史节点后，前端优先通过 HTTP 历史接口拉取完整消息列表并直接渲染。
- 如需“逐块回放动画”或断线恢复流式状态，可选通过 ACP `session/load` 获取块级历史。
- 设计约束: 历史查询属于数据读取链路，不依赖 Rust；实时对话和工具调用才依赖 Rust-ACP 通道。

### 5.3 模块 C: 全局分叉（Fork）

#### 5.3.1 一键 Fork
- 在会话页提供“一键 Fork”入口。
- 新会话默认复制从根到当前叶子的完整历史链路。

#### 5.3.2 状态继承
- 子会话继承父会话的必要运行上下文（如工作目录、已激活技能集合）。
- 子会话后续消息与父会话必须物理隔离存储。

### 5.4 模块 D: POI 断点派生

#### 5.4.1 原子标注
- Agent 输出渲染时，段落级别绑定可追踪 `messageId`。

#### 5.4.2 交互入口
- 用户选中回复片段后，右键菜单显示“以此为断点派生”。

#### 5.4.3 断点 Fork 流程
- 前端提取选区对应 `messageId`。
- Rust 通过 ACP 发送 `session/fork(sessionId, messageId)`。
- 服务端截断到该断点并创建新会话，返回新 `sessionId`。

## 6. 安全与人机协同

### 6.1 能力边界（Capabilities）
- 路径沙箱: 仅允许访问用户授权目录及其子路径。
- 命令黑名单: 对高风险命令模式（如 `rm -rf`、危险 SQL）进行拦截。
- 最小权限原则: 未声明能力默认拒绝。

### 6.2 审批工作流（HITL）
- 以下操作必须进入审批: 文件写入、终端执行、跨目录访问。
- Rust 向前端发出 `session/request_permission` 事件并携带完整参数。
- 用户批准后执行并回传结果，拒绝时返回 `permission_denied` 给服务端。

### 6.3 审计与可追踪性
- 记录审批日志: 请求参数、审批人操作、时间戳、执行结果。
- 日志需按会话可检索，用于问题追溯。

## 7. 接口草案

### 7.1 HTTP API（Node）
- `GET /api/sessions`: 获取会话树。
- `GET /api/sessions/:id/history`: 获取指定会话历史消息（支持分页/游标）。
- `PATCH /api/sessions/:id`: 重命名会话。
- `DELETE /api/sessions/:id`: 删除会话节点（可配置是否级联）。
- `GET /api/settings`: 获取模型与应用配置。

### 7.2 ACP 消息（Rust <-> Node）
- 会话类: `session/start`, `session/prompt`, `session/load`, `session/fork`。
- 工具类: `fs/read_text_file`, `fs/write_text_file`, `terminal/create`。
- 流式类: `agent_message_chunk`（含 `text_delta`, `thinking_delta`）。
- 权限类: `session/request_permission`, `permission/approve`, `permission/reject`。

## 8. 非功能需求

### 8.1 性能
- 首屏会话树加载时间目标: 小于 1.5 秒（1k 节点以内）。
- 首 token 可见时间目标: 小于 2 秒（网络正常条件）。

### 8.2 稳定性
- ACP 连接断开后应自动重连，并保证会话状态可恢复。
- 工具调用失败必须有错误码与可读错误信息。

### 8.3 可维护性
- 前后端协议字段要有版本标识。
- 关键链路（握手、工具调用、审批）必须具备结构化日志。

## 9. 验收标准（MVP Definition of Done）

### 9.1 功能验收
- 能完成至少 10 轮连续对话，流式消息展示正常。
- Agent 可在授权目录内成功读取本地文件并用于回答。
- 终端命令可执行并实时回传输出。
- 写操作存在审批弹窗，拒绝后不落盘。
- 会话树可展示、切换、回放历史。
- 全局 Fork 与 POI Fork 均可成功创建新分支并继续对话。

### 9.2 安全验收
- 越权路径访问会被拒绝并记录日志。
- 命令黑名单命中时必须触发人工审批或直接拒绝。

### 9.3 质量验收
- 核心链路具备基础自动化测试: 会话 API、审批流程、路径校验。
- 无阻断级崩溃问题，关键错误可追踪。

## 10. 里程碑计划

### Phase 1: 协议贯通
- 完成 Node ACP WebSocket Server 与 Rust ACP Client 握手。
- 建立最小对话闭环（prompt -> stream -> render）。

### Phase 2: 环境委派
- 打通 `fs/read_text_file` 与 `terminal/create`。
- 完成路径沙箱与审批基础框架。

### Phase 3: 树状管理
- 完成 HTTP 会话树 API 与前端目录树渲染。
- 支持 `session/load` 历史回放。

### Phase 4: 分叉增强
- 上线全局 Fork。
- 实现基于 `messageId` 的 POI 断点 Fork。
- 完成端到端验收与问题收敛。

## 11. 风险与缓解

- 风险: WebSocket 连接抖动导致对话中断。
     缓解: Rust 侧连接状态机 + 自动重连 + 消息幂等处理。
- 风险: 本地命令执行能力引入安全隐患。
     缓解: 能力白名单、命令黑名单、强制审批。
- 风险: 会话树与消息历史一致性问题。
     缓解: 服务端统一写入顺序与版本号校验。

## 12. 原型设计（Wireframes）

基于需求文档的 UI 界面原型与交互说明。

### 12.1 全局布局（主界面）

采用经典的 C/S 桌面应用布局（如类似 Cursor / Chat 界面），左侧为会话树管理，右侧为对话与操控区。

```text
+-------------------------------------------------------------------------+
|  [+] 新建会话树 (New)                           [⚙️ 本地模型/环境设置]  |
+--------------------------+----------------------------------------------+
| 🌲 会话树 (Session Tree)  | 💬 聊天界面 (Chat Area)                      |
|                          |                                              |
| 🔍 搜索历史会话...        | [Agent] 10:42 AM                             |
|                          | ▶ 思考过程 (Thinking...) [点击展开]          |
| 🔽 默认工作区             | ------------------------------------------   |
|  ├── 📄 初始化项目        | 我已经为您读取了 `server/index.ts`。         |
|  │   ├── 📄 增加API      | 基于您的需求，建议对 WebSocket 握手进行以下修改：|
|  │   │   └── 📄 调试Bug  | ...                                          |
|  │   └── 📄 重构代码 (当前)|                                              |
|  ├── 📄 Rust ACP 联调     | [User] 10:45 AM                              |
|                          | 那就按照建议修改吧，注意兜底异常。           |
|                          |                                              |
|                          | [Agent] 10:45 AM                             |
|                          | ⏳ 正在请求执行文件写入操作...               |
|                          | [ ✓ 文件已修改 ]                             |
|                          |                                              |
|                          |                        [🔀 一键从当前派生]   |
|                          +----------------------------------------------+
|                          | 📝 告诉 Agent 下一步做什么...                |
|                          | [ 📎 添加上下文 ]                 [ 发送 ↗ ] |
+--------------------------+----------------------------------------------+
```

### 12.2 核心功能与交互说明

#### 12.2.1 会话树与分支管理（Tree Management）
- **展示**：左侧边栏以树状结构（父子关系）展示所有的历史对话，每个节点包含 `title` 和 `timestamp`。
- **当前定位**：当前正在进行的会话分支会有明显的高亮选中态。
- **一键 Fork（全局派生）**：在聊天界面底部，提供「一键从当前派生」按钮。点击后，基于当前完整的链路（从根节点到当前叶子），复制出一个新的子节点，并且在此独立沙箱中继续对话。

#### 12.2.2 消息流与流式渲染（Stream & Thinking）
- **增量渲染**：对话内容必须支持 `text_delta` 打字机渲染，首 token 响应 < 2s。
- **思考折叠（`thinking_delta`）**：在普通文字输出前，展示 Agent 的推理和规划过程。界面默认将其装载在可折叠的容器中（`▶ 思考过程`），保持主界面干净，用户可点击展开排障。

#### 12.2.3 POI 断点派生（POI Fork）
当用户对 Agent 的某段特有回答（或代码片段）有疑问，想专门针对这句话展开新话题，而不引起原会话的主线发散时：

```text
+------------------------------------------------------------+
| [Agent]                                                    |
| 推荐边界: 历史数据读取优先走 HTTP；仅实时对话走 ACP...     |
|                                                            |
|      +------------------------------------------+          |
|      | 🖱️ 选中句子后右键 / 悬浮菜单              |          |
|      | ---------------------------------------- |          |
|      | 📋 复制 (Copy)                           |          |
|      | 🔀 以此为断点派生 (Fork from this point)  | <------- |
|      | 📌 设为书签 (Bookmark)                    |          |
|      +------------------------------------------+          |
+------------------------------------------------------------+
```
- **交互**：选中片段（POI）或在对应消息气泡上右键，弹出菜单。选择「以此为断点派生」，前端提取包含该文本的 `messageId`，通过 ACP 请求服务端将其作为新会话树的起点，界面自动切换至新建的树节点。

#### 12.2.4 人机协作授权审批弹窗（HITL Auth Modal）
当核心大脑（服务端）通过 ACP 请求执行本地高风险操作（写文件、执行 Shell 命令）时，Tauri (Rust) 强制拦截并弹出系统级/应用级模态审批窗口：

**终端命令审批**：
```text
+-------------------------------------------------------------+
| ⚠️ 本地操作授权请求 (Action Required)                         |
|-------------------------------------------------------------|
| Agent 正在请求访问您的本地系统以执行操作，请确认：              |
|                                                             |
| 🚨 操作类型：终端命令执行 (terminal/create)                   |
| 💻 请求命令：`rm -rf ./temp_build`                            |
| 📂 工作目录：/Users/zhiyu/Desktop/coding/divisor-agent/      |
|                                                             |
| 风险提示：此命令匹配到高风险操作黑名单，执行后不可撤销。        |
|                                                             |
| [ ❌ 拒绝 (Reject) ]                   [ ✅ 批准 (Approve) ] |
+-------------------------------------------------------------+
```

**文件修改审批**：
```text
+-------------------------------------------------------------+
| ⚠️ 本地文件写入审批                                           |
|-------------------------------------------------------------|
| 📂 目标路径：.../divisor-agent/packages/server/src/index.ts  |
|                                                             |
| 📄 差异预览 (Diff)：                                         |
| ---------------------------------------------------------   |
|   // original                                               |
| - const wsPort = 8080;                                      |
|   // modified                                               |
| + const wsPort = process.env.VITE_WS_PORT || 8080;          |
| ---------------------------------------------------------   |
|                                                             |
| [ ❌ 拒绝写入 ]                               [ ✅ 允许写入 ] |
+-------------------------------------------------------------+
```

- **审批反馈**：
  - 点击"批准"：触发 IPC 授信给 Rust 执行命令或落盘操作，结果反馈给 Node。
  - 点击"拒绝"：Rust 返回 `permission_denied` 给 Node，并在聊天流中展示出"用户已拒绝该操作"。
